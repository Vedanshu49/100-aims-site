## Internal Page Links

**Description**

Internal page links use the `id` attribute to link to specific sections within a page or to other pages on your site. This allows users to jump directly to relevant content without scrolling through entire pages.

### Key Concepts

- **id attribute:** Unique identifier for elements
- **Fragment identifier:** The # symbol in URLs
- **Same-page links:** Jump to sections on current page
- **Cross-page links:** Jump to sections on other pages
- **Semantic:** Use meaningful IDs

### Same-Page Links

```html
<!-- Link to section -->
<a href="#about">Jump to About</a>

<!-- The target section -->
<h2 id="about">About Us</h2>
<p>Our story...</p>
```

### Table of Contents

```html
<nav>
  <h2>Contents</h2>
  <ul>
    <li><a href="#intro">Introduction</a></li>
    <li><a href="#methods">Methods</a></li>
    <li><a href="#results">Results</a></li>
    <li><a href="#conclusion">Conclusion</a></li>
  </ul>
</nav>

<section id="intro">
  <h2>Introduction</h2>
  <p>Content...</p>
</section>

<section id="methods">
  <h2>Methods</h2>
  <p>Content...</p>
</section>

<section id="results">
  <h2>Results</h2>
  <p>Content...</p>
</section>

<section id="conclusion">
  <h2>Conclusion</h2>
  <p>Content...</p>
</section>
```

### Cross-Page Links

```html
<!-- Link to section on another page -->
<a href="/blog/article.html#section1">Read Section 1</a>

<!-- Link to home page with anchor -->
<a href="/#contact">Contact Us</a>
```

### ID Best Practices

```html
<!-- ✅ Good: Descriptive, lowercase, hyphens -->
<h2 id="getting-started">Getting Started</h2>
<div id="faq-section">FAQ</div>
<p id="important-note">Note: Read carefully</p>

<!-- ❌ Bad: Unclear, spaces, uppercase -->
<h2 id="GettingStarted">Getting Started</h2>
<div id="faq section">FAQ</div>
<p id="note1">Note</p>
```

### Real-World Examples

**FAQ Page:**
```html
<nav>
  <a href="#faq-html">HTML FAQ</a>
  <a href="#faq-css">CSS FAQ</a>
  <a href="#faq-js">JavaScript FAQ</a>
</nav>

<section id="faq-html">
  <h2>HTML Frequently Asked Questions</h2>
  <!-- FAQ content -->
</section>

<section id="faq-css">
  <h2>CSS Frequently Asked Questions</h2>
  <!-- FAQ content -->
</section>

<section id="faq-js">
  <h2>JavaScript Frequently Asked Questions</h2>
  <!-- FAQ content -->
</section>
```

**Back to Top Link:**
```html
<div id="top">
  <!-- Page content -->
</div>

<footer>
  <a href="#top">Back to Top</a>
</footer>
```

### Common Mistakes

❌ **Missing id attribute:**
```html
<a href="#section">Jump</a>
<h2>Section</h2>  <!-- No id -->
```
✅ **Add matching id:**
```html
<a href="#section">Jump</a>
<h2 id="section">Section</h2>
```

❌ **Duplicate ids:**
```html
<h2 id="title">First Title</h2>
<h2 id="title">Second Title</h2>  <!-- Duplicate id! -->
```
✅ **Unique ids:**
```html
<h2 id="title-1">First Title</h2>
<h2 id="title-2">Second Title</h2>
```

### ID Naming Conventions

```html
<!-- Use lowercase and hyphens -->
<div id="main-content">Content</div>
<section id="featured-products">Products</section>
<nav id="top-navigation">Nav</nav>

<!-- Avoid underscores and spaces -->
<div id="main_content">Avoid this</div>
<div id="main content">Avoid this</div>
```

### Practice Tips

1. Use meaningful, descriptive id names
2. Keep ids lowercase
3. Use hyphens to separate words
4. Ensure ids are unique on each page
5. Test all links work correctly
6. Use for accessibility and navigation
