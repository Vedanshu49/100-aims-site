## Heading Tags (h1-h6)

**Description**

Headings are used to structure and organize content on a webpage. HTML provides six heading levels (`<h1>` through `<h6>`) that create a hierarchy, with `<h1>` being the most important and `<h6>` being the least important. Proper heading structure is crucial for readability, accessibility, and SEO.

### Key Concepts

- **Hierarchy:** H1 is most important, H6 is least important
- **Structure:** Headings organize content logically
- **Accessibility:** Screen readers use headings for navigation
- **SEO:** Search engines use headings to understand page content
- **Semantics:** Headings convey meaning, not just size

### All Six Heading Levels

```html
<h1>This is a heading 1 (most important)</h1>
<h2>This is a heading 2</h2>
<h3>This is a heading 3</h3>
<h4>This is a heading 4</h4>
<h5>This is a heading 5</h5>
<h6>This is a heading 6 (least important)</h6>
```

### Visual Appearance

In a browser, headings appear progressively smaller:

```
This is a heading 1 (most important)     ← Largest
This is a heading 2
This is a heading 3
This is a heading 4
This is a heading 5
This is a heading 6 (least important)    ← Smallest
```

### Typical Document Structure

```html
<!DOCTYPE html>
<html>
<body>
  <!-- One H1 per page - the main title -->
  <h1>My Blog</h1>
  
  <!-- H2 for main sections -->
  <h2>Article 1: Learning HTML</h2>
  <p>Article content...</p>
  
  <!-- H3 for subsections within H2 -->
  <h3>Getting Started</h3>
  <p>Content about getting started...</p>
  
  <h3>Best Practices</h3>
  <p>Content about best practices...</p>
  
  <!-- Next H2 section -->
  <h2>Article 2: CSS Styling</h2>
  <p>Article content...</p>
  
  <h3>Selectors</h3>
  <p>Content about selectors...</p>
</body>
</html>
```

### Why Heading Hierarchy Matters

**✅ Good Structure (Logical Hierarchy):**
```html
<h1>Product Guide</h1>           ← Main title
  <h2>Getting Started</h2>       ← Main section
    <h3>Installation</h3>        ← Subsection
    <h3>Configuration</h3>       ← Subsection
  <h2>Advanced Features</h2>     ← Main section
    <h3>Customization</h3>       ← Subsection
```

**❌ Bad Structure (Skips Levels):**
```html
<h1>Product Guide</h1>
  <h3>Getting Started</h3>       ← Jumps from H1 to H3 (skip H2!)
  <h5>Installation</h5>          ← Jumps to H5
```

### Screen Reader Experience

Screen readers announce headings to help users navigate. A logical structure helps:

```
Screen reader output:
- Heading 1: Product Guide
  - Heading 2: Getting Started
    - Heading 3: Installation
    - Heading 3: Configuration
  - Heading 2: Advanced Features
    - Heading 3: Customization
```

### Accessibility Best Practices

```html
<!-- ✅ Good: One H1 per page -->
<h1>Home Page</h1>
<section>
  <h2>Featured Products</h2>
  <h3>Product A</h3>
  <h3>Product B</h3>
</section>

<!-- ❌ Bad: Multiple H1s or no clear structure -->
<h1>Welcome</h1>
<h1>Featured Items</h1>
<h1>About Us</h1>
```

### SEO Implications

Search engines use heading structure to understand page content:

```html
<h1>Best Python Learning Resources</h1>      ← Main topic
<h2>Online Courses</h2>                       ← Subtopic
<h3>Free Courses</h3>
<h3>Premium Courses</h3>

<h2>Books for Learning Python</h2>           ← Another subtopic
<h3>Beginner Books</h3>
<h3>Advanced Books</h3>
```

### Styling Headings with HTML

Headings have default styling, but you can change it with CSS:

```html
<!-- CSS will style these differently -->
<h1>Large Title</h1>
<h2>Medium Subtitle</h2>
<h3>Small Section Title</h3>

<!-- Style sheet (CSS) -->
<style>
  h1 { color: blue; font-size: 2.5rem; }
  h2 { color: green; font-size: 2rem; }
  h3 { color: orange; font-size: 1.5rem; }
</style>
```

### Don't Use Headings for Styling

**❌ Wrong: Using heading for size only**
```html
<!-- Just because H1 is bigger... -->
<h1>Regular paragraph text that should be <p> element</h1>
```

**✅ Right: Use semantic elements**
```html
<!-- Use heading for hierarchy, <p> for text -->
<h1>Page Title</h1>
<p>Regular paragraph text.</p>
```

### Common Mistakes

❌ **Multiple H1s on same page:**
```html
<h1>Home</h1>
<h1>Products</h1>
<h1>About</h1>
```
✅ **One H1, rest are H2+:**
```html
<h1>Main Page Title</h1>
<h2>Products Section</h2>
<h2>About Section</h2>
```

❌ **Skipping levels:**
```html
<h1>Title</h1>
<h3>Subtitle</h3>  ← Should be H2
```
✅ **Sequential levels:**
```html
<h1>Title</h1>
<h2>Subtitle</h2>
```

❌ **Using headings just for size:**
```html
<h1>Making this text BIG</h1>  ← Wrong reason
```
✅ **Use for proper hierarchy:**
```html
<h1>Document Title</h1>        ← Right reason
<p>Regular text can be styled with CSS for size</p>
```

### Real-World Usage

- All professional websites use proper heading structure
- Blog posts use H1 for article title, H2 for sections
- Documentation sites use headings for organization
- News websites use headings for story hierarchy
- E-commerce sites use headings for product categories

### Practice Tips

1. **One H1 per page:** Plan your page structure before coding
2. **Logical sequence:** Don't skip heading levels
3. **Use headings for structure:** Not just to make text big
4. **Test with screen reader:** Understand the user experience
5. **Check SEO tools:** Many tools check heading structure
6. **Read your outline:** Your headings should form a good outline
