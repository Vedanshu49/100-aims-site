## Open Links in New Tab

**Description**

By default, links open in the same tab, replacing the current page. Using the `target="_blank"` attribute allows you to open links in a new tab or window. This is useful for external links so users don't lose their place on your site.

### Key Concepts

- **target="_blank":** Opens link in new tab
- **rel attribute:** Security consideration for external links
- **User Experience:** Keeps your site visible while viewing external content
- **Mobile vs Desktop:** Behaves slightly differently on mobile
- **Accessibility:** Consider user preferences

### Basic Syntax

```html
<a href="https://example.com" target="_blank">
  Open in New Tab
</a>
```

### Complete Example

```html
<!-- External link opening in new tab -->
<a href="https://wikipedia.org" target="_blank">
  Wikipedia
</a>

<!-- Internal link (usually same tab) -->
<a href="/about">About</a>

<!-- External link in new tab with security -->
<a href="https://github.com" target="_blank" rel="noopener noreferrer">
  GitHub
</a>
```

### Security: The rel Attribute

When using `target="_blank"`, always add `rel="noopener noreferrer"`:

```html
<!-- ✅ Secure way -->
<a href="https://external.com" target="_blank" rel="noopener noreferrer">
  External Site
</a>

<!-- ❌ Insecure (vulnerable) -->
<a href="https://external.com" target="_blank">
  External Site
</a>
```

**Why?** Prevents the opened page from accessing `window.opener` and accessing your site.

### Target Values

```html
<!-- _blank: New tab/window -->
<a href="https://example.com" target="_blank">New Tab</a>

<!-- _self: Same tab (default) -->
<a href="/page" target="_self">Same Tab</a>

<!-- _parent: Parent frame -->
<a href="/page" target="_parent">Parent</a>

<!-- _top: Top-level frame -->
<a href="/page" target="_top">Top</a>
```

### Real-World Examples

**Navigation Bar:**
```html
<nav>
  <a href="/">Home</a>
  <a href="/about">About</a>
  <a href="https://twitter.com/mysite" target="_blank" rel="noopener noreferrer">
    Twitter
  </a>
</nav>
```

**External Resources:**
```html
<p>
  Learn more from
  <a href="https://mdn.org" target="_blank" rel="noopener noreferrer">
    MDN Web Docs
  </a>.
</p>
```

**Social Media Links:**
```html
<footer>
  <a href="https://facebook.com/mysite" target="_blank" rel="noopener noreferrer">
    Facebook
  </a>
  <a href="https://twitter.com/mysite" target="_blank" rel="noopener noreferrer">
    Twitter
  </a>
  <a href="https://instagram.com/mysite" target="_blank" rel="noopener noreferrer">
    Instagram
  </a>
</footer>
```

### When to Use target="_blank"

**✅ Use For:**
- External websites
- Documentation links
- Social media links
- Downloadable PDFs
- Third-party tools

**❌ Don't Use For:**
- Internal navigation
- Links within your site
- When user experience requires staying on your site

### Common Mistakes

❌ **Missing rel attribute:**
```html
<a href="https://external.com" target="_blank">Link</a>
```
✅ **Include rel for security:**
```html
<a href="https://external.com" target="_blank" rel="noopener noreferrer">Link</a>
```

❌ **Using for internal links:**
```html
<a href="/about" target="_blank">About</a>
```
✅ **Keep internal links in same tab:**
```html
<a href="/about">About</a>
```

### Styling Open-in-New-Tab Links

```html
<style>
  a[target="_blank"]::after {
    content: " ↗";  /* External link icon */
  }
</style>

<a href="https://example.com" target="_blank">External</a>
```

### Accessibility Considerations

Inform users when links open in new tabs:

```html
<!-- Explicit indication -->
<a href="https://example.com" target="_blank">
  Example (opens in new tab)
</a>

<!-- Aria-label for screen readers -->
<a href="https://example.com" target="_blank" aria-label="Example external site opens in new window">
  Example
</a>
```

### Practice Tips

1. **Use target="_blank" for external links**
2. **Always add rel="noopener noreferrer"**
3. **Inform users when links open new tabs**
4. **Test in different browsers**
5. **Consider mobile behavior**
