## Table Headers

**Description**

Table headers (`<th>`) identify column and row headings. They're semantically different from data cells (`<td>`) and improve accessibility for screen readers. Headers can be in the first row, first column, or both.

### Column Headers

```html
<table>
  <tr>
    <th>Product</th>
    <th>Price</th>
    <th>Stock</th>
  </tr>
  <tr>
    <td>Laptop</td>
    <td>$999</td>
    <td>15</td>
  </tr>
</table>
```

### Row Headers

```html
<table>
  <tr>
    <th>Monday</th>
    <td>9:00 AM - 5:00 PM</td>
  </tr>
  <tr>
    <th>Tuesday</th>
    <td>9:00 AM - 5:00 PM</td>
  </tr>
  <tr>
    <th>Wednesday</th>
    <td>Closed</td>
  </tr>
</table>
```

### Scope Attribute

```html
<table>
  <tr>
    <th scope="col">Name</th>
    <th scope="col">Age</th>
  </tr>
  <tr>
    <td>John</td>
    <td>30</td>
  </tr>
</table>
```

The `scope` attribute helps screen readers understand header relationships:
- `scope="col"` - header for a column
- `scope="row"` - header for a row

### Real-World Example

```html
<table>
  <tr>
    <th scope="col">Month</th>
    <th scope="col">Revenue</th>
    <th scope="col">Expenses</th>
  </tr>
  <tr>
    <th scope="row">January</th>
    <td>$50,000</td>
    <td>$30,000</td>
  </tr>
  <tr>
    <th scope="row">February</th>
    <td>$55,000</td>
    <td>$32,000</td>
  </tr>
</table>
```

### Practice Tips

1. Always use th for headers
2. Use scope attribute for clarity
3. Place headers first
4. Keep headers concise
5. Use for accessibility
