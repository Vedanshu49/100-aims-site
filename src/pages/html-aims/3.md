## HTML Elements & Tags

**Description**

HTML elements and tags are the building blocks of web pages. Understanding the difference between them and how they work is fundamental to HTML. A **tag** is the code you write, while an **element** is what the tag represents in the page.

### Key Concepts

- **Tag:** The code written in angle brackets like `<p>`
- **Element:** The complete unit including opening tag, content, and closing tag
- **Attributes:** Additional information provided to tags
- **Nesting:** Elements can contain other elements
- **Semantics:** Tags have meaning—they describe their content

### Tags vs. Elements

**Tag:**
```html
<p>   ← This is a tag (just the code)
</p>  ← This is also a tag
```

**Element:**
```html
<p>This is my paragraph.</p>  ← This entire thing is an element
```

### Anatomy of an HTML Element

```html
<p class="intro">This is a paragraph.</p>
│ │   │   │    │                       │
│ │   │   │    └─ Content
│ │   │   └─ Attribute value
│ │   └─ Attribute name
│ └─ Opening tag
└─ Element name (p = paragraph)
                                      │
                                      └─ Closing tag
```

### Common Elements You'll Use

```html
<!-- Headings -->
<h1>Main Title</h1>
<h2>Subtitle</h2>

<!-- Paragraphs -->
<p>This is text</p>

<!-- Line break -->
<br>  ← No closing tag needed

<!-- Horizontal rule -->
<hr>  ← No closing tag needed

<!-- Images -->
<img src="photo.jpg" alt="My photo">

<!-- Links -->
<a href="https://google.com">Click here</a>

<!-- Buttons -->
<button>Click Me</button>

<!-- Lists -->
<ul>
  <li>Item 1</li>
  <li>Item 2</li>
</ul>
```

### Element Categories

**Block Elements:** Take up full width, start on new line
```html
<p>Paragraph</p>
<h1>Heading</h1>
<div>Container</div>
```

**Inline Elements:** Only take up necessary width, flow with text
```html
<span>Inline text</span>
<a>Link</a>
<strong>Bold text</strong>
```

**Attributes:** Provide additional information to tags
```html
<!-- class attribute (for styling) -->
<p class="highlight">Important</p>

<!-- id attribute (unique identifier) -->
<div id="header">Top section</div>

<!-- data-* attributes (custom data) -->
<div data-user-id="123">User</div>
```

### Nesting: Elements Inside Elements

```html
<div>
  <p>This is a paragraph <strong>with bold text</strong> inside.</p>
  <p>This is another paragraph.</p>
</div>
```

**Proper nesting:** Inner elements close before outer elements close

### Empty vs. Content Elements

**Elements with Content:**
```html
<p>Text goes inside</p>
<h1>Heading text</h1>
```

**Self-closing (Empty Elements):**
```html
<br>        ← Line break (no content)
<hr>        ← Horizontal rule (no content)
<img>       ← Image (no content)
<input>     ← Form input (no content)
```

### Common Mistakes

❌ **Incorrect nesting:**
```html
<p>Text <strong>emphasized</p></strong>  ← Wrong order!
```
✅ **Correct nesting:**
```html
<p>Text <strong>emphasized</strong></p>
```

❌ **Forgetting to close tags:**
```html
<p>Paragraph
<p>Another
```
✅ **Always close tags:**
```html
<p>Paragraph</p>
<p>Another</p>
```

❌ **Self-closing tags with slash:**
```html
<img src="photo.jpg" />  ← Unnecessary in HTML5
```
✅ **HTML5 way:**
```html
<img src="photo.jpg">
```

### Real-World Usage

- Professional websites use 30+ different HTML elements
- Semantic elements improve accessibility and SEO
- Proper nesting helps with CSS styling
- Understanding elements is crucial for web development

### Practice Tips

1. **Memorize common elements:** p, div, span, h1-h6, a, img
2. **Practice nesting:** Create nested divs with paragraphs inside
3. **Inspect real websites:** View source code to see how professionals use elements
4. **Validate:** Use https://validator.w3.org/ to check your HTML
5. **Create examples:** Write HTML for different scenarios
