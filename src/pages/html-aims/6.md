## HTML Comments

**Description**

Comments are notes you add to your HTML code that the browser doesn't display. They're invisible to website visitors but visible to anyone reading your code. Comments help you remember why you wrote something and help other developers understand your code.

### Key Concepts

- **Invisible:** Comments don't appear on the webpage
- **Readable:** Visible to developers viewing the source code
- **Explanations:** Help document your code
- **Temporary Notes:** Good for marking sections you're working on
- **Best Practice:** Use for complex logic or important notes

### Comment Syntax

```html
<!-- This is a comment -->
```

**Structure:**
```html
<!--  SPACE  Comment text  SPACE  -->
│      │                              │
│      │                              └─ End delimiter
│      └─ Space (recommended)
└─ Start delimiter
```

### Single-Line Comments

```html
<!-- This is a single-line comment -->

<!-- Mark the header section -->
<header>
  <h1>My Website</h1>
</header>

<!-- Navigation menu -->
<nav>
  <a href="/">Home</a>
  <a href="/about">About</a>
</nav>
```

### Multi-Line Comments

```html
<!--
  This is a multi-line comment.
  It spans across multiple lines.
  You can write as much as you need.
-->

<!--
  Main header section
  Contains site title and tagline
  Updated: Nov 15, 2025
-->
<header>
  <h1>My Website</h1>
  <p>Welcome to my site!</p>
</header>
```

### Commenting Out Code (Temporarily Hide Elements)

```html
<!-- Temporarily hidden while testing -->
<!-- <a href="/about">About</a> -->

<!-- Can comment out multiple lines -->
<!--
<section>
  <h2>Featured Posts</h2>
  <article>Post content</article>
</section>
-->

<!-- Use this temporarily -->
<section>
  <h2>New Featured Posts</h2>
  <article>Updated post</article>
</section>
```

### Comments in Document Sections

```html
<!DOCTYPE html>
<html>
<head>
  <!-- Meta tags for browser settings -->
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  
  <!-- Page title (appears in browser tab) -->
  <title>My Website</title>
  
  <!-- Link to CSS stylesheet -->
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <!-- Header section with navigation -->
  <header>
    <h1>My Website</h1>
  </header>
  
  <!-- Main content area -->
  <main>
    <p>Welcome!</p>
  </main>
  
  <!-- Footer with links -->
  <footer>
    <p>&copy; 2025 My Website</p>
  </footer>
</body>
</html>
```

### When to Use Comments

**✅ Good Use Cases:**
```html
<!-- Section divider -->
<!-- TODO: Add user profile functionality -->
<!-- FIXME: This needs to be optimized -->
<!-- Complex CSS class names require explanation -->
<div class="grid-col-3-md-2-sm-1-xs">
  Content
</div>
```

**❌ Avoid Over-Commenting:**
```html
<!-- Heading -->
<h1>Hello</h1>

<!-- Paragraph -->
<p>Text</p>

<!-- Link -->
<a href="/">Home</a>
```

### Common Comment Patterns

**Section Headers:**
```html
<!-- ============== NAVIGATION SECTION ============== -->
<nav>
  ...
</nav>

<!-- ============== MAIN CONTENT ============== -->
<main>
  ...
</main>
```

**TODO Markers:**
```html
<!-- TODO: Add form validation -->
<form>
  ...
</form>

<!-- FIXME: Fix mobile layout on small screens -->
<div class="container">
  ...
</div>
```

**Notes for Other Developers:**
```html
<!-- NOTE: This class is used by JavaScript in script.js -->
<button class="send-notification">
  Send
</button>

<!-- DEPRECATED: Use new-feature instead -->
<!-- <div class="old-feature"></div> -->
```

### Common Mistakes

❌ **Missing closing comment:**
```html
<!-- This comment never closes
<p>Text</p>
```
✅ **Always close comments:**
```html
<!-- This comment closes properly -->
<p>Text</p>
```

❌ **Comments inside comments (doesn't work):**
```html
<!--
  <!-- This nested comment breaks -->
-->
```
✅ **Single comment level:**
```html
<!-- Single level comment -->
```

❌ **Dashes in middle without spacing:**
```html
<!--This is a comment-with-dashes-->
```
✅ **Proper spacing:**
```html
<!-- This is a comment with dashes -->
```

### Keyboard Shortcuts

**Most editors support comment shortcuts:**

```
Windows/Linux: Ctrl + /
Mac: Cmd + /
```

Highlight code and press the shortcut to auto-comment/uncomment!

### Real-World Usage

- Professional developers comment complex code
- Teams use comments to communicate
- Comments help with code maintenance
- TODO comments track work to be done
- Comments are helpful during debugging

### Privacy Note

**Important:** Comments are visible to anyone viewing your page source.

```html
<!-- NEVER put sensitive info in comments -->
<!-- DON'T: Password is admin123 -->
<!-- DON'T: API key is xyz789 -->
<!-- DON'T: Database username is db_user -->
```

Sensitive information should NEVER go in comments or any HTML code!

### Practice Tips

1. **Comment your sections:** Mark major sections with comments
2. **Explain the why:** Not just what, but why you did something
3. **Use shortcuts:** Learn your editor's comment keyboard shortcut
4. **Keep them updated:** Update comments if you change code
5. **Don't over-comment:** Code should be self-explanatory where possible
6. **Remove old comments:** Clean up commented-out code before deploying
