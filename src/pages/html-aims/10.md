## Text Formatting (strong, em)

**Description**

HTML provides semantic tags for emphasizing and highlighting text: `<strong>` for important text and `<em>` for emphasized text. These elements don't just make text look different—they add meaning to your content, which helps with accessibility and SEO. Understanding the semantic difference between formatting elements is crucial.

### Key Concepts

- **Semantic:** Tags convey meaning, not just styling
- **`<strong>`:** Indicates strong importance
- **`<em>`:** Indicates emphasis or stress
- **Accessibility:** Screen readers recognize and announce emphasis
- **Distinction:** Don't confuse with `<b>` and `<i>` tags

### The STRONG Tag

The `<strong>` tag indicates that text is **strongly important**:

```html
<p>
  <strong>Warning:</strong> Do not touch the red button!
</p>

<p>
  The deadline is <strong>November 15th, 2025</strong>.
</p>

<p>
  <strong>Important:</strong> This section requires your attention.
</p>
```

**Display:** Text appears **bold** in browsers

### The EM Tag

The `<em>` tag indicates that text is *emphasized* or has special stress:

```html
<p>
  You <em>must</em> read the documentation.
</p>

<p>
  This is a <em>really</em> important concept.
</p>

<p>
  Let me <em>repeat</em> this crucial point.
</p>
```

**Display:** Text appears *italic* in browsers

### Multiple Emphases

```html
<p>
  This is <strong><em>very important</em></strong> information.
</p>

<p>
  You <em>must</em> <strong>complete</strong> this task <strong>immediately</strong>.
</p>
```

### Screen Reader Behavior

When a screen reader encounters these elements, it conveys the meaning:

**STRONG tag:**
```html
<strong>Important warning</strong>
```
Screen reader: "Important warning" (with emphasized tone)

**EM tag:**
```html
<em>Please read</em> this carefully
```
Screen reader: "Please read" (with different intonation)

### STRONG vs. B Tag

**`<strong>` (Semantic - Use This):**
```html
<!-- Indicates importance -->
<p><strong>Caution:</strong> Wet floor!</p>
```

**`<b>` (Generic - Avoid):**
```html
<!-- Just makes text bold, no meaning -->
<p><b>Caution:</b> Wet floor!</p>
```

The `<strong>` tag is better because it adds *meaning* to your content.

### EM vs. I Tag

**`<em>` (Semantic - Use This):**
```html
<!-- Emphasizes with stress -->
<p>You <em>must</em> pay attention!</p>
```

**`<i>` (Generic - Avoid):**
```html
<!-- Just makes text italic, no meaning -->
<p>You <i>must</i> pay attention!</p>
```

The `<em>` tag is better because it adds *stress* to your content.

### Real-World Examples

**Warning Messages:**
```html
<p>
  <strong>IMPORTANT:</strong> This action cannot be undone. 
  Please proceed with <strong>caution</strong>.
</p>
```

**Instructions:**
```html
<p>
  Step 1: <strong>Do not</strong> turn off the power during installation.
</p>

<p>
  Step 2: <strong>Ensure</strong> all cables are properly connected.
</p>

<p>
  Step 3: <strong>Wait</strong> for the installation to complete.
</p>
```

**Emphasis in Dialogue:**
```html
<p>
  "I <em>really</em> need your help with this!"
</p>

<p>
  "You <strong>must</strong> be careful with this task."
</p>
```

**Article with Formatting:**
```html
<h1>How to Cook the Perfect Steak</h1>

<p>
  The key to a perfect steak is <strong>temperature control</strong>. 
  You <em>must</em> let your steak come to room temperature before cooking.
</p>

<h2>Temperature Guide</h2>

<p>
  <strong>Rare:</strong> 120-130°F<br>
  <strong>Medium Rare:</strong> 130-135°F (recommended)<br>
  <strong>Medium:</strong> 135-145°F<br>
  <strong>Well Done:</strong> 155°F+
</p>

<p>
  <em>Never</em> skip the resting period after cooking.
</p>
```

### Common Mistakes

❌ **Using B and I instead of STRONG and EM:**
```html
<p>This is <b>bold</b> and this is <i>italic</i>.</p>
```
✅ **Use semantic tags:**
```html
<p>This is <strong>important</strong> and this is <em>emphasized</em>.</p>
```

❌ **Nesting in wrong order:**
```html
<p><em><strong>Text</em></strong></p>
```
✅ **Proper nesting:**
```html
<p><strong><em>Text</em></strong></p>
```

❌ **Using for styling only:**
```html
<p>
  Normal text <strong>Larger text (just for size)</strong> More text
</p>
```
✅ **Use for semantic meaning:**
```html
<p>
  Normal text <strong>Important information</strong> More text
</p>
```

❌ **Overusing emphasis:**
```html
<p>
  This is <strong><em>super</em></strong> <strong><em>duper</em></strong> 
  <strong><em>important</em></strong> text.
</p>
```
✅ **Use emphasis appropriately:**
```html
<p>
  This is <strong>very important</strong> text.
</p>
```

### Combining with Other Elements

```html
<!-- In paragraphs -->
<p>
  The <strong>deadline</strong> is <em>absolutely critical</em> for project success.
</p>

<!-- In lists -->
<ul>
  <li><strong>Priority 1:</strong> Complete the setup</li>
  <li><strong>Priority 2:</strong> Test all features</li>
  <li><strong>Priority 3:</strong> Document the process</li>
</ul>

<!-- In links -->
<p>
  Click here to read the <strong><a href="/rules">important rules</a></strong>.
</p>

<!-- In headings -->
<h2>Why <em>This</em> Matters</h2>
```

### Styling with CSS

```html
<style>
  strong {
    color: red;           /* Make strong text red */
    font-weight: bold;    /* Ensure it's bold */
  }
  
  em {
    color: blue;          /* Make em text blue */
    font-style: italic;   /* Ensure it's italic */
  }
</style>

<p>This is <strong>important</strong> and <em>emphasized</em>.</p>
```

### Accessibility Impact

Screen reader users rely on these semantic tags:

```html
<!-- Screen readers announce this clearly -->
<p><strong>Deadline:</strong> November 15th</p>

<!-- This just shows styling to sighted users -->
<p><b>Deadline:</b> November 15th</p>
```

### When to Use Each

**Use `<strong>` for:**
- Important information
- Warnings and cautions
- Critical deadlines
- Required actions
- Strong emphasis

**Use `<em>` for:**
- Words requiring stress
- Quotations with emphasis
- Important distinctions
- Emotional emphasis
- Reading emphasis

### Real-World Usage

- News articles use `<strong>` for breaking news
- Legal documents use `<strong>` for important terms
- Instructions use `<strong>` for critical steps
- Quotes use `<em>` for emphasis
- Technical docs use `<strong>` for warnings

### Practice Tips

1. **Prefer semantic tags:** Always use `<strong>` and `<em>` over `<b>` and `<i>`
2. **Use purposefully:** Don't overuse emphasis
3. **Match meaning:** Use strong for importance, em for stress
4. **Test with screen readers:** Verify accessibility
5. **Combine appropriately:** Multiple emphases are okay if meaningful
6. **Style with CSS:** Don't rely on default styling
