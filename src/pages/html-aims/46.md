## Accessibility Best Practices

**Description**

Web accessibility ensures websites are usable by everyone, including people with disabilities. Follow WCAG guidelines to make your site accessible to screen readers, keyboard navigation, and assistive technologies.

### Semantic HTML

```html
<!-- Good: Semantic -->
<header>
  <nav>Navigation</nav>
</header>
<main>
  <article>Content</article>
</main>

<!-- Bad: Only div -->
<div class="header">
  <div class="nav">Navigation</div>
</div>
```

### Labels for Form Elements

```html
<!-- Always use labels -->
<label for="email">Email Address:</label>
<input type="email" id="email" name="email">

<!-- Not this -->
<input type="email" placeholder="Email Address">
```

### Alt Text for Images

```html
<!-- Good: Descriptive alt text -->
<img src="dog.jpg" alt="Golden retriever playing in park">

<!-- Bad: Empty or vague -->
<img src="dog.jpg" alt="image">
<img src="dog.jpg">
```

### Heading Structure

```html
<!-- Good: Logical hierarchy -->
<h1>Page Title</h1>
<h2>Main Section</h2>
<h3>Subsection</h3>
<h2>Another Section</h2>

<!-- Bad: Skipping levels -->
<h1>Title</h1>
<h3>Subsection</h3>
```

### Keyboard Navigation

```html
<!-- Provide focus states -->
<style>
  button:focus-visible {
    outline: 3px solid blue;
  }
  
  a:focus-visible {
    outline: 2px solid blue;
  }
</style>

<!-- Ensure all elements are reachable via Tab -->
<button>Clickable</button>
<a href="#">Link</a>
<input type="text">
```

### Color Contrast

```html
<!-- Good contrast ratio (WCAG AA) -->
<p style="color: #000; background: #fff;">Text</p>

<!-- Bad: Insufficient contrast -->
<p style="color: #999; background: #eee;">Harder to read</p>
```

### ARIA Attributes

```html
<!-- Provide additional context -->
<button aria-label="Close menu">Ã—</button>

<div role="alert" aria-live="polite">
  Error message here
</div>

<img src="icon.svg" alt="Success">
```

### Real-World Examples

**Accessible Form:**
```html
<form>
  <fieldset>
    <legend>Contact Information</legend>
    
    <label for="name">Name:</label>
    <input type="text" id="name" name="name" required>
    
    <label for="email">Email:</label>
    <input type="email" id="email" name="email" required>
    
    <label for="message">Message:</label>
    <textarea id="message" name="message" required></textarea>
    
    <button type="submit">Submit</button>
  </fieldset>
</form>
```

**Accessible Navigation:**
```html
<nav aria-label="Main navigation">
  <ul>
    <li><a href="/">Home</a></li>
    <li><a href="/about">About</a></li>
    <li><a href="/contact">Contact</a></li>
  </ul>
</nav>
```

### WCAG Guidelines

1. **Perceivable** - Content visible to all senses
2. **Operable** - Keyboard accessible, enough time
3. **Understandable** - Clear language, predictable
4. **Robust** - Works with assistive technology

### Practice Tips

1. Use semantic HTML
2. Always use alt text
3. Provide labels
4. Maintain heading hierarchy
5. Test with screen readers
6. Ensure keyboard navigation
7. Check color contrast
8. Avoid color alone for meaning
