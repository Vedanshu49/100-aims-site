## HTML Document Structure

**Description**

Every HTML document follows a standard structure. This structure is like a template that organizes your content into sections, making it easier for browsers to understand and display your website correctly. Understanding this structure is crucial before writing any HTML.

### Key Concepts

- **DOCTYPE Declaration:** Tells the browser which version of HTML you're using
- **HTML Root Element:** Wraps the entire document
- **Head Section:** Contains metadata and configuration
- **Body Section:** Contains all visible content
- **Hierarchical Structure:** Elements are nested within each other

### Complete HTML Document Structure

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My Website</title>
</head>
<body>
  <h1>Welcome to My Website</h1>
  <p>This is my first website.</p>
</body>
</html>
```

### Explanation of Each Part

**Line 1: DOCTYPE Declaration**
```html
<!DOCTYPE html>
```
- Declares that this document uses HTML5 (the current standard)
- Must be the first line of an HTML document
- Not actually an HTML element, just an instruction for the browser

**Lines 2-3: Opening HTML Tag**
```html
<html lang="en">
```
- Root element that wraps all other elements
- `lang="en"` specifies the page language (helpful for screen readers and translation)

**Lines 4-8: Head Section**
```html
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My Website</title>
</head>
```
- **`<meta charset="UTF-8">`** - Specifies character encoding (supports all languages)
- **`<meta name="viewport">`** - Makes site responsive on mobile devices
- **`<title>`** - Appears in browser tab and search results

**Lines 9-12: Body Section**
```html
<body>
  <h1>Welcome to My Website</h1>
  <p>This is my first website.</p>
</body>
```
- Contains all visible content
- Everything users see on the page goes here

### Minimal vs. Complete Structure

**Minimal (Still Works):**
```html
<!DOCTYPE html>
<html>
<body>
  <p>Hello!</p>
</body>
</html>
```

**Complete (Best Practice):**
```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Page Title</title>
</head>
<body>
  <p>Hello!</p>
</body>
</html>
```

### Common Mistakes

❌ **Missing DOCTYPE:**
```html
<html>
  <!-- Browser might use quirks mode -->
</html>
```
✅ **Always include it:**
```html
<!DOCTYPE html>
<html>
```

❌ **Content in Head:**
```html
<head>
  <p>This should not be visible</p>
</head>
```
✅ **Content goes in Body:**
```html
<body>
  <p>This is visible</p>
</body>
```

### Important Meta Tags

```html
<head>
  <!-- Character encoding (supports international characters) -->
  <meta charset="UTF-8">
  
  <!-- Responsive design for mobile devices -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  
  <!-- Page description for search engines -->
  <meta name="description" content="This is my website">
  
  <!-- Prevents search engines from indexing -->
  <meta name="robots" content="noindex">
</head>
```

### Real-World Usage

- **Every website** follows this structure
- **Single Page Applications** (React, Vue) generate this structure dynamically
- **Server-side rendering** produces this structure on the server
- **Content Management Systems** automatically create this structure
- **Email templates** use similar structure for emails

### Practice Tips

1. **Use a template:** Copy the complete structure above for all your HTML files
2. **Set the language:** Always use `lang="en"` or appropriate language code
3. **Check the title:** Change the `<title>` for each page—it matters for SEO
4. **Test responsiveness:** Remove the viewport meta tag and see how pages look on mobile
5. **Validate structure:** Use https://validator.w3.org/ to check for errors
