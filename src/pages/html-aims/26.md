## When to Use Tables

**Description**

Tables should only be used for displaying tabular data (information organized in rows and columns). Never use tables for page layout. They're semantically meaningful for data but create accessibility issues when misused.

### Correct Use Cases

**Data Table:**
```html
<table>
  <tr>
    <th>Name</th>
    <th>Age</th>
    <th>City</th>
  </tr>
  <tr>
    <td>John</td>
    <td>28</td>
    <td>New York</td>
  </tr>
</table>
```

**Pricing Table:**
```html
<table>
  <tr>
    <th>Plan</th>
    <th>Price</th>
    <th>Features</th>
  </tr>
  <tr>
    <td>Starter</td>
    <td>$9/mo</td>
    <td>5 projects</td>
  </tr>
</table>
```

### Incorrect Use

❌ **Using tables for layout:**
```html
<!-- WRONG! -->
<table>
  <tr>
    <td><!-- Navigation --></td>
    <td><!-- Content --></td>
    <td><!-- Sidebar --></td>
  </tr>
</table>
```

✅ **Use CSS Grid or Flexbox for layout:**
```html
<!-- RIGHT! -->
<div style="display: grid; grid-template-columns: 1fr 3fr 1fr;">
  <nav>Navigation</nav>
  <main>Content</main>
  <aside>Sidebar</aside>
</div>
```

### When NOT to Use Tables

- Page layouts
- Navigation menus
- Decorative spacing
- Form layouts
- Content organization (use div + CSS)

### Practice Tips

1. Ask: "Is this data tabular?"
2. If no, use other HTML elements
3. Tables are for data only
4. Use semantic HTML otherwise
5. Tables affect accessibility
