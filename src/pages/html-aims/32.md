## Form Action & Method

**Description**

The `action` attribute specifies where form data is sent. The `method` attribute specifies how it's sent. Choose between GET (visible in URL) and POST (hidden, secure) based on your needs.

### GET Method

```html
<form action="/search" method="GET">
  <input type="search" name="q">
  <button type="submit">Search</button>
</form>
```

Results in URL: `/search?q=searchterm`

### POST Method

```html
<form action="/submit" method="POST">
  <input type="email" name="email">
  <button type="submit">Subscribe</button>
</form>
```

Data sent in request body (not visible in URL)

### GET vs POST

| Aspect | GET | POST |
|--------|-----|------|
| Visibility | Data in URL | Data hidden |
| Security | Less secure | More secure |
| Use Case | Search, filters | Forms, login |
| Size Limit | Limited | Larger data |

### Real-World Examples

**Search Form (GET):**
```html
<form action="/products" method="GET">
  <input type="text" name="search" placeholder="Search products">
  <select name="category">
    <option>All Categories</option>
    <option>Electronics</option>
  </select>
  <button type="submit">Search</button>
</form>
```

**Login Form (POST):**
```html
<form action="/login" method="POST">
  <input type="email" name="email" required>
  <input type="password" name="password" required>
  <button type="submit">Login</button>
</form>
```

### Practice Tips

1. Use GET for search/filtering
2. Use POST for sensitive data
3. Always validate server-side
4. Use HTTPS for secure forms
5. Include CSRF protection
