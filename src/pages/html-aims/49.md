## HTML Validation

**Description**

HTML validation checks your code for syntax errors and adherence to HTML standards. Use the W3C Validator or other tools to find and fix issues. Valid HTML ensures better browser compatibility, accessibility, and performance.

### Why Validate HTML?

1. **Browser Compatibility** - Works across all browsers
2. **Accessibility** - Screen readers work properly
3. **SEO** - Search engines prefer valid HTML
4. **Performance** - Cleaner code loads faster
5. **Maintenance** - Easier to debug and update

### Using W3C Validator

**Online Tool:**
- Visit: https://validator.w3.org/
- Paste HTML or upload file
- View errors and warnings
- Fix issues and revalidate

**Common Errors:**

```html
<!-- Error: Unclosed tag -->
<p>This paragraph is not closed

<!-- Error: Nested incorrectly -->
<p>Text <h2>Heading</h2></p>

<!-- Error: Invalid attribute -->
<div invalid-attr="value">Text</div>

<!-- Error: Missing required attribute -->
<img src="image.jpg">  <!-- Missing alt -->

<!-- Error: Duplicate id -->
<div id="main">Content</div>
<div id="main">More content</div>
```

### Valid HTML Example

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Valid HTML Page</title>
</head>
<body>
  <header>
    <h1>Welcome</h1>
    <nav>
      <ul>
        <li><a href="/">Home</a></li>
        <li><a href="/about">About</a></li>
      </ul>
    </nav>
  </header>
  
  <main>
    <article>
      <h2>Article Title</h2>
      <p>Article content with proper closing tags.</p>
      <img src="photo.jpg" alt="Description of photo">
    </article>
  </main>
  
  <footer>
    <p>&copy; 2024. All rights reserved.</p>
  </footer>
</body>
</html>
```

### Common Validation Issues

| Issue | Example | Fix |
|-------|---------|-----|
| Unclosed tags | `<p>Text` | `<p>Text</p>` |
| Missing alt | `<img src="a.jpg">` | `<img src="a.jpg" alt="desc">` |
| Invalid nesting | `<p><div></div></p>` | `<div><p></p></div>` |
| Duplicate IDs | `id="main"` twice | Use unique IDs |
| Missing DOCTYPE | N/A | `<!DOCTYPE html>` |

### Validation Tools

1. **W3C Validator** - https://validator.w3.org/
2. **HTML5 Validator** - Online and CLI tools
3. **Browser DevTools** - Built-in validation
4. **IDE Extensions** - Real-time checking
5. **GitHub Actions** - Automated validation

### Real-World Example

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="description" content="Contact us page">
  <title>Contact Us</title>
</head>
<body>
  <header>
    <h1>Contact</h1>
  </header>
  
  <main>
    <form action="/submit" method="POST">
      <label for="name">Name:</label>
      <input type="text" id="name" name="name" required>
      
      <label for="email">Email:</label>
      <input type="email" id="email" name="email" required>
      
      <label for="message">Message:</label>
      <textarea id="message" name="message" rows="5"></textarea>
      
      <button type="submit">Send</button>
    </form>
  </main>
  
  <footer>
    <p>&copy; 2024</p>
  </footer>
</body>
</html>
```

### Practice Tips

1. Validate regularly during development
2. Fix errors as they appear
3. Use DOCTYPE declaration
4. Close all tags properly
5. Use meaningful IDs/classes
6. Test in multiple browsers
7. Keep HTML semantic
8. Use a linter in your editor
