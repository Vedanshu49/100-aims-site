## Form Container

**Description**

The `<form>` element is the container for all form inputs. It establishes how data is collected and submitted. Every form needs attributes like `action` (where to send data) and `method` (how to send it).

### Basic Form Structure

```html
<form>
  <label for="name">Name:</label>
  <input type="text" id="name" name="name">
  
  <label for="email">Email:</label>
  <input type="email" id="email" name="email">
  
  <button type="submit">Submit</button>
</form>
```

### Form with Action and Method

```html
<form action="/submit" method="POST">
  <input type="text" name="username" placeholder="Username">
  <input type="password" name="password" placeholder="Password">
  <button type="submit">Login</button>
</form>
```

### Real-World Examples

**Contact Form:**
```html
<form action="/contact" method="POST">
  <input type="text" name="name" placeholder="Your Name" required>
  <input type="email" name="email" placeholder="Your Email" required>
  <textarea name="message" placeholder="Your Message"></textarea>
  <button type="submit">Send Message</button>
</form>
```

**Search Form:**
```html
<form action="/search" method="GET">
  <input type="search" name="q" placeholder="Search...">
  <button type="submit">Search</button>
</form>
```

### Common Mistakes

❌ **Missing form element:**
```html
<input type="text" name="username">
<input type="submit">
```

✅ **Wrap inputs in form:**
```html
<form>
  <input type="text" name="username">
  <button type="submit">Submit</button>
</form>
```

### Practice Tips

1. Always use form element
2. Set action and method
3. Use POST for sensitive data
4. Use GET for searches
5. Validate server-side
