## Void/Empty Elements

**Description**

Some HTML elements don't have a closing tag because they don't contain any content. These are called **void elements** or **empty elements**. They're self-contained units that don't need text or other elements inside them. Understanding which elements are void is important for writing correct HTML.

### Key Concepts

- **Void Elements:** Don't have a closing tag
- **Content:** These elements cannot contain any content
- **Self-Contained:** They represent a single thing (image, line break, etc.)
- **Common Examples:** `<img>`, `<br>`, `<hr>`, `<input>`
- **No Closing Slash:** HTML5 doesn't use `/>` syntax

### Common Void Elements

| Element | Purpose | Example |
|---------|---------|---------|
| `<img>` | Embed an image | `<img src="photo.jpg">` |
| `<br>` | Line break | `<br>` |
| `<hr>` | Horizontal rule | `<hr>` |
| `<input>` | Form input field | `<input type="text">` |
| `<meta>` | Metadata | `<meta charset="UTF-8">` |
| `<link>` | Link external resource | `<link rel="stylesheet">` |
| `<area>` | Image map area | `<area shape="rect">` |
| `<base>` | Document base URL | `<base href="/">` |
| `<col>` | Table column | `<col>` |
| `<embed>` | External content | `<embed src="video.mp4">` |
| `<source>` | Media source | `<source src="video.mp4">` |
| `<track>` | Subtitle/caption track | `<track kind="captions">` |
| `<wbr>` | Word break opportunity | `Text<wbr>Break` |

### Images (Most Common Void Element)

```html
<!-- Image with required attributes -->
<img src="photo.jpg" alt="My photo">

<!-- Image with optional attributes -->
<img src="photo.jpg" alt="My photo" width="200" height="150">

<!-- These are WRONG (don't do this) -->
<img src="photo.jpg"></img>     ← Unnecessary closing tag
<img src="photo.jpg" />          ← Old XHTML style (not needed in HTML5)
<img src="photo.jpg">            ← Missing alt attribute (bad for accessibility)
```

### Line Breaks

```html
<!-- Single line break -->
<p>Line 1<br>Line 2</p>

<!-- Multiple line breaks -->
<p>
  Line 1<br>
  Line 2<br>
  Line 3
</p>

<!-- Not this (too much) -->
<p><br><br><br>Too many line breaks</p>
```

### Horizontal Rule

```html
<!-- Creates a visual line across the page -->
<hr>

<!-- Useful for separating sections -->
<section>First section</section>
<hr>
<section>Second section</section>
```

### Form Inputs (Void Elements)

```html
<!-- Text input -->
<input type="text" placeholder="Enter name">

<!-- Password input -->
<input type="password">

<!-- Email input -->
<input type="email">

<!-- Number input -->
<input type="number">

<!-- Checkbox -->
<input type="checkbox">

<!-- Radio button -->
<input type="radio">
```

### Metadata Void Elements

```html
<!-- In the head section of your document -->
<head>
  <!-- Character encoding -->
  <meta charset="UTF-8">
  
  <!-- Viewport for responsive design -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  
  <!-- Page description -->
  <meta name="description" content="My website">
  
  <!-- Link to CSS stylesheet -->
  <link rel="stylesheet" href="style.css">
</head>
```

### HTML5 vs. XHTML Syntax

**Old XHTML Style (with slash):**
```html
<img src="photo.jpg" />
<br />
<hr />
<input type="text" />
```

**Modern HTML5 Style (without slash):**
```html
<img src="photo.jpg">
<br>
<hr>
<input type="text">
```

Use HTML5 syntax—it's the current standard!

### Common Mistakes

❌ **Adding a closing tag to void elements:**
```html
<img src="photo.jpg"></img>  ← Wrong!
<br></br>                    ← Wrong!
<hr></hr>                    ← Wrong!
```
✅ **Correct (no closing tags):**
```html
<img src="photo.jpg">
<br>
<hr>
```

❌ **Missing required attributes:**
```html
<img>  ← Missing src attribute
<input>  ← Missing type attribute (sometimes)
```
✅ **With required attributes:**
```html
<img src="photo.jpg" alt="Photo">
<input type="text">
```

❌ **Using old XHTML syntax:**
```html
<br />
<img src="photo.jpg" />
```
✅ **Using HTML5 syntax:**
```html
<br>
<img src="photo.jpg">
```

### Void vs. Non-Void Comparison

```html
<!-- Non-void (has opening and closing tags) -->
<p>Paragraph content</p>
<div>Div content</div>
<span>Span content</span>

<!-- Void (no closing tag, no content) -->
<img src="photo.jpg">
<br>
<hr>
<input type="text">
```

### Real-World Usage

- Images use `<img>` element (void)
- Line breaks use `<br>` element (void)
- Form inputs use `<input>` element (void)
- Metadata in head uses void elements
- Videos use `<source>` element (void)

### Practice Tips

1. **Memorize common void elements:** img, br, hr, input
2. **No closing tag:** If an element is void, don't close it
3. **Alt attributes:** Always use alt text on images for accessibility
4. **Validate:** Use https://validator.w3.org/ to check your HTML
5. **Test images:** Make sure images display correctly in your page
