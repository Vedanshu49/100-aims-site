## Input Validation Attributes

**Description**

HTML5 provides built-in validation attributes that check input values before submission. These include `required`, `min`, `max`, `minlength`, `maxlength`, and `pattern`. Browser validates automatically on submit.

### Required Attribute

```html
<input type="text" name="username" required>
<input type="email" name="email" required>
```

### Min and Max

```html
<!-- For numbers -->
<label for="age">Age:</label>
<input type="number" id="age" name="age" min="0" max="120">

<!-- For date -->
<label for="deadline">Deadline:</label>
<input type="date" id="deadline" name="deadline" min="2024-01-01">
```

### Length Validation

```html
<!-- Text length -->
<label for="username">Username (3-20 chars):</label>
<input type="text" id="username" name="username" minlength="3" maxlength="20">

<!-- Password strength -->
<label for="password">Password (min 8 chars):</label>
<input type="password" id="password" name="password" minlength="8">
```

### Pattern Validation

```html
<!-- Phone format -->
<label for="phone">Phone (xxx-xxx-xxxx):</label>
<input type="tel" id="phone" name="phone" pattern="[0-9]{3}-[0-9]{3}-[0-9]{4}">

<!-- Alphanumeric only -->
<input type="text" name="code" pattern="[A-Za-z0-9]+">

<!-- Postal code -->
<input type="text" name="zip" pattern="[0-9]{5}">
```

### Real-World Examples

**Registration Form:**
```html
<form action="/register" method="POST">
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required>
  
  <label for="password">Password:</label>
  <input type="password" id="password" name="password" minlength="8" required>
  
  <label for="confirm">Confirm Password:</label>
  <input type="password" id="confirm" name="confirm" minlength="8" required>
  
  <label for="age">Age:</label>
  <input type="number" id="age" name="age" min="18" max="120" required>
  
  <button type="submit">Register</button>
</form>
```

### Important Notes

1. **Browser validates** - Prevents submission if invalid
2. **Server validate too** - Always validate server-side
3. **HTML5 only** - Older browsers may not support
4. **Pattern attribute** - Uses regex for complex rules

### Practice Tips

1. Use appropriate type
2. Add validation attributes
3. Combine multiple validations
4. Always validate server-side
5. Provide error messages
