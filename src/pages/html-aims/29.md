## Table Semantic Structure

**Description**

Use semantic table elements (`<thead>`, `<tbody>`, `<tfoot>`) to structure tables logically. This improves accessibility, styling, and maintainability.

### Table Sections

```html
<table>
  <thead>
    <tr>
      <th>Name</th>
      <th>Age</th>
      <th>City</th>
    </tr>
  </thead>
  
  <tbody>
    <tr>
      <td>Alice</td>
      <td>28</td>
      <td>NYC</td>
    </tr>
    <tr>
      <td>Bob</td>
      <td>35</td>
      <td>LA</td>
    </tr>
  </tbody>
  
  <tfoot>
    <tr>
      <th colspan="3">2 records total</th>
    </tr>
  </tfoot>
</table>
```

### Element Definitions

- `<thead>` - Table header containing column titles
- `<tbody>` - Main table data (can have multiple)
- `<tfoot>` - Table footer (summaries, totals)

### Complete Example

```html
<table>
  <thead>
    <tr>
      <th>Product</th>
      <th>Q1</th>
      <th>Q2</th>
      <th>Q3</th>
    </tr>
  </thead>
  
  <tbody>
    <tr>
      <td>Laptops</td>
      <td>1,200</td>
      <td>1,400</td>
      <td>1,600</td>
    </tr>
    <tr>
      <td>Phones</td>
      <td>2,000</td>
      <td>2,300</td>
      <td>2,500</td>
    </tr>
  </tbody>
  
  <tfoot>
    <tr>
      <th>Total</th>
      <td>3,200</td>
      <td>3,700</td>
      <td>4,100</td>
    </tr>
  </tfoot>
</table>
```

### Accessibility Benefits

- Screen readers understand structure
- Better organization
- Easier to style with CSS
- Improved maintainability

### Practice Tips

1. Always use thead/tbody/tfoot
2. Keep footers for summaries
3. Use scope attributes with headers
4. Test with screen readers
5. Validate semantics
