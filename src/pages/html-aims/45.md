## Semantic HTML5 Tags

**Description**

Semantic HTML tags communicate meaning about content to browsers, screen readers, and developers. Tags like `<article>`, `<section>`, `<nav>`, `<header>`, `<footer>`, and `<aside>` describe content purpose, not just presentation.

### Common Semantic Tags

```html
<!-- Page structure -->
<header>
  <nav>Navigation here</nav>
</header>

<main>
  <article>Main content</article>
  <aside>Sidebar</aside>
</main>

<footer>Footer information</footer>
```

### Article vs Section

```html
<!-- Article: Self-contained content -->
<article>
  <h2>Blog Post Title</h2>
  <p>Blog content...</p>
</article>

<!-- Section: Thematic grouping -->
<section>
  <h2>Chapter 1</h2>
  <p>Chapter content...</p>
</section>
```

### Complete Page Structure

```html
<!DOCTYPE html>
<html>
<head>
  <title>My Website</title>
</head>
<body>
  <!-- Page header with navigation -->
  <header>
    <h1>Website Title</h1>
    <nav>
      <a href="/">Home</a>
      <a href="/about">About</a>
      <a href="/contact">Contact</a>
    </nav>
  </header>
  
  <!-- Main content -->
  <main>
    <article>
      <h2>Blog Post</h2>
      <p>Content...</p>
    </article>
    
    <aside>
      <h3>Related Links</h3>
      <ul>
        <li><a href="#">Link 1</a></li>
        <li><a href="#">Link 2</a></li>
      </ul>
    </aside>
  </main>
  
  <!-- Footer -->
  <footer>
    <p>&copy; 2024. All rights reserved.</p>
  </footer>
</body>
</html>
```

### Semantic Tags

- `<header>` - Introductory content
- `<nav>` - Navigation links
- `<main>` - Primary content (one per page)
- `<article>` - Self-contained content
- `<section>` - Thematic grouping
- `<aside>` - Related content, sidebar
- `<footer>` - Closing information

### Real-World Example

```html
<header>
  <h1>Tech Blog</h1>
  <nav>
    <a href="/">Home</a>
    <a href="/articles">Articles</a>
    <a href="/about">About</a>
  </nav>
</header>

<main>
  <article>
    <h2>Getting Started with HTML</h2>
    <p>Published: Jan 15, 2024</p>
    <section>
      <h3>Introduction</h3>
      <p>Introduction text...</p>
    </section>
    <section>
      <h3>Basics</h3>
      <p>Basics text...</p>
    </section>
  </article>
  
  <aside>
    <h3>Popular Posts</h3>
    <ul>
      <li><a href="#">CSS Tips</a></li>
      <li><a href="#">JavaScript Basics</a></li>
    </ul>
  </aside>
</main>

<footer>
  <p>&copy; 2024 Tech Blog</p>
</footer>
```

### Benefits

1. **Accessibility** - Screen readers understand structure
2. **SEO** - Search engines understand content
3. **Maintainability** - Code is self-documenting
4. **Semantics** - Meaning is clear to developers
5. **Future-proof** - Browsers optimize semantic tags

### Practice Tips

1. Use semantic tags when applicable
2. Use div for grouping only
3. Maintain proper nesting
4. One main per page
5. Test with screen readers
