## Href Attribute

**Description**

The `href` attribute is the most important part of the anchor tag. It specifies where a link goes. `href` stands for "hypertext reference" and can point to different types of destinations: other web pages, files, email addresses, phone numbers, or sections within the same page.

### Key Concepts

- **href:** The attribute that defines the link destination
- **Required:** The href attribute is required for functional links
- **Absolute URLs:** Full web address including protocol
- **Relative URLs:** Path to file relative to current page
- **Various destinations:** Web pages, files, email, phone, sections

### Absolute URLs (Full Address)

Absolute URLs include the complete address starting with the protocol:

```html
<!-- HTTP (not secure) -->
<a href="http://example.com">Example</a>

<!-- HTTPS (secure - preferred) -->
<a href="https://www.example.com">Example</a>

<!-- With www prefix -->
<a href="https://www.google.com">Google</a>

<!-- Without www prefix -->
<a href="https://github.com">GitHub</a>
```

### Relative URLs (Relative Path)

Relative URLs are paths relative to the current page:

```html
<!-- Same directory -->
<a href="about.html">About</a>

<!-- Subdirectory -->
<a href="products/shoes.html">Shoes</a>

<!-- Parent directory -->
<a href="../index.html">Home</a>

<!-- Root of website -->
<a href="/">Home</a>

<!-- Nested path -->
<a href="/blog/posts/first-post.html">First Post</a>
```

### Protocol-Relative URLs

URLs that work with both HTTP and HTTPS:

```html
<!-- Works with http:// or https:// -->
<a href="//cdn.example.com/image.jpg">CDN Image</a>
```

### Different Href Destinations

**Web Page:**
```html
<a href="https://example.com">Example</a>
```

**HTML File in Subdirectory:**
```html
<a href="/about/team.html">Our Team</a>
```

**PDF Document:**
```html
<a href="/downloads/guide.pdf">Download Guide</a>
```

**Image File:**
```html
<a href="/images/photo.jpg">View Photo</a>
```

**Email Address:**
```html
<a href="mailto:contact@example.com">Email Us</a>
```

**Phone Number:**
```html
<a href="tel:+1-555-123-4567">Call Us</a>
```

**Page Anchor (Section):**
```html
<a href="#section1">Jump to Section 1</a>
```

### Complex Href Examples

**Email with Subject:**
```html
<a href="mailto:info@example.com?subject=Question">Email Us</a>
```

**Email with Subject and Body:**
```html
<a href="mailto:info@example.com?subject=Inquiry&body=Hello%20there">Email</a>
```

**Phone with Extension:**
```html
<a href="tel:+1-555-123-4567x123">Call Extension 123</a>
```

**Link to External Site:**
```html
<a href="https://www.wikipedia.org">Wikipedia</a>
```

### File Paths and Href

**Directory Structure:**
```
website/
├── index.html
├── about.html
├── contact.html
└── products/
    ├── shoes.html
    └── hats.html
```

**Links from index.html:**
```html
<!-- Link to about.html (same directory) -->
<a href="about.html">About</a>

<!-- Link to products/shoes.html (subdirectory) -->
<a href="products/shoes.html">Shoes</a>
```

**Links from products/shoes.html:**
```html
<!-- Link to index.html (parent directory) -->
<a href="../index.html">Home</a>

<!-- Link to hats.html (same directory) -->
<a href="hats.html">Hats</a>

<!-- Link to about.html (parent directory) -->
<a href="../about.html">About</a>
```

### Absolute vs. Relative: When to Use

**Use Absolute URLs for:**
```html
<!-- External websites -->
<a href="https://google.com">Google</a>

<!-- When certainty is needed -->
<a href="https://example.com/about">About Us</a>

<!-- API endpoints -->
<a href="https://api.example.com/data">Data</a>
```

**Use Relative URLs for:**
```html
<!-- Internal navigation -->
<a href="/about">About</a>

<!-- Same folder links -->
<a href="page.html">Page</a>

<!-- Subdirectory links -->
<a href="products/list.html">Products</a>

<!-- Parent directory -->
<a href="../index.html">Home</a>
```

### Real-World Examples

**Navigation Menu with Relative URLs:**
```html
<nav>
  <a href="/">Home</a>
  <a href="/products">Products</a>
  <a href="/about">About</a>
  <a href="/contact">Contact</a>
</nav>
```

**Blog Post with Links:**
```html
<article>
  <h1>My Blog Post</h1>
  <p>
    Learn more about
    <a href="/html-tutorial">HTML</a> and
    <a href="/css-tutorial">CSS</a>.
  </p>
  <a href="https://mdn.org" target="_blank">
    Official Documentation
  </a>
</article>
```

**Footer Links:**
```html
<footer>
  <a href="/">Home</a>
  <a href="/privacy">Privacy</a>
  <a href="/terms">Terms</a>
  <a href="https://twitter.com/mysite" target="_blank">Twitter</a>
</footer>
```

### Common Mistakes

❌ **Forgetting href attribute:**
```html
<a>Link text</a>  ← No href, not clickable
```
✅ **Always include href:**
```html
<a href="/page">Link text</a>
```

❌ **Incorrect relative path:**
```html
<!-- From products/shoes.html -->
<a href="about.html">About</a>  ← Wrong! File is in parent directory
```
✅ **Correct relative path:**
```html
<!-- From products/shoes.html -->
<a href="../about.html">About</a>  ← Correct!
```

❌ **Missing protocol for external links:**
```html
<a href="example.com">Example</a>  ← Browser might not recognize
```
✅ **Include protocol:**
```html
<a href="https://example.com">Example</a>
```

❌ **Absolute path for internal links:**
```html
<!-- On example.com/about -->
<a href="/home">Home</a>  ← Points to root, might not be what you want
```
✅ **Relative path often better:**
```html
<a href="/">Home</a>  ← Clear relative path
```

### Special Characters in URLs

Some characters need encoding in href:

```html
<!-- Space (should be %20 or +) -->
<a href="/search?q=html%20tutorial">Search</a>

<!-- Ampersand -->
<a href="/page?a=1&b=2">Link</a>

<!-- Hash for anchors -->
<a href="#section1">Jump</a>
```

### Testing Links

**Check these:**
1. Does the link work?
2. Does it go to the right place?
3. Does relative path work from all pages?
4. Are external links using https?
5. Do links open correctly in new tabs?

### Real-World Usage

- All websites use href attributes
- Navigation depends on correct hrefs
- SEO depends on proper link structure
- Relative paths make sites portable
- Absolute URLs point to external resources

### Practice Tips

1. **Test all links:** Click every link to verify it works
2. **Use relative paths:** For internal links (more portable)
3. **Use absolute URLs:** For external links (more reliable)
4. **Include protocol:** Always use https:// for external links
5. **Check paths:** Verify relative paths are correct
6. **Avoid broken links:** Check that files exist at specified paths
7. **Use consistent structure:** Organize files logically for easier paths
